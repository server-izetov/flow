# Read Before Asserting

Before stating a fact about how the system works, name the file you
read **this session** that verifies it. No such read → it is a guess:
either read first, or label it explicitly ("haven't read this, my
guess is…") and then read before acting on it.

Any of these framings require a current-session read first:

- "X is caused by Y" / "the reason is Y" / "found it — it's Y"
- "X works by Y" / "Y is what's happening here"
- A hedged guess ("likely", "probably", "almost certainly") — still a guess
- Citing a `.claude/rules/` mechanism as the explanation: the rule names a
  class; the instance is verified only by reading the code that produces it
- Explaining user-provided diagnostic output (CI failure, stack trace) —
  read the named file/function before explaining it
